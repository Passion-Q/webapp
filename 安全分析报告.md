### 2.1 密码安全

#### 2.1.1 BCrypt 密码加密

**安全准则**：密码绝不以明文形式存储，采用强哈希算法进行单向加密。

**代码实现**：

```java
// SecurityConfig.java - 配置密码加密器
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

// UserService.java - 用户注册时加密密码
@Transactional
public User register(RegisterRequest request) {
    // ...验证逻辑...
    User user = new User();
    user.setPassword(passwordEncoder.encode(request.getPassword()));  // BCrypt加密
    return userRepository.save(user);
}
```

**安全原理**：

- BCrypt 采用 Blowfish 算法，内置盐值（salt），防止彩虹表攻击
- 每次加密生成不同的哈希值，即使相同密码也无法识别
- 计算成本可配置，抵御暴力破解攻击

#### 2.1.2 密码复杂度校验

**安全准则**：强制要求高复杂度密码，提高账户安全性。

**代码实现**：

```java
// PasswordComplexityValidator.java - 自定义密码复杂度校验器
public class PasswordComplexityValidator implements ConstraintValidator<PasswordComplexity, String> {
    private static final String UPPERCASE_REGEX = ".*[A-Z].*";      // 大写字母
    private static final String LOWERCASE_REGEX = ".*[a-z].*";      // 小写字母
    private static final String DIGIT_REGEX = ".*[0-9].*";          // 数字
    private static final String SPECIAL_CHAR_REGEX = ".*[!@#$%^&*()_+\\-=\\[\\]{};':\"\\\\|,.<>/?].*";  // 特殊字符

    @Override
    public boolean isValid(String password, ConstraintValidatorContext context) {
        boolean hasUppercase = password.matches(UPPERCASE_REGEX);
        boolean hasLowercase = password.matches(LOWERCASE_REGEX);
        boolean hasDigit = password.matches(DIGIT_REGEX);
        boolean hasSpecialChar = password.matches(SPECIAL_CHAR_REGEX);

        if (!hasUppercase || !hasLowercase || !hasDigit || !hasSpecialChar) {
            context.buildConstraintViolationWithTemplate(message.toString()).addConstraintViolation();
            return false;
        }
        return true;
    }
}
```

**使用方式**：

```java
// RegisterRequest.java - DTO 中使用自定义注解
public class RegisterRequest {
    @NotBlank(message = "密码不能为空")
    @Size(min = 8, max = 100, message = "密码长度必须在8-100之间")
    @PasswordComplexity  // 自定义复杂度校验注解
    private String password;
}
```

**安全效果**：

- 强制密码至少 8 位
- 必须包含大写字母、小写字母、数字、特殊字符
- 暴力破解难度显著提升

#### 2.1.3 修改密码功能

**安全准则**：修改密码必须验证旧密码，防止账户被盗后密码被篡改。

**代码实现**：

```java
// UserService.java - 修改密码方法（第93-108行）
@Transactional
public void changePassword(String username, String oldPassword, String newPassword) {
    User user = userRepository.findByUsername(username)
            .orElseThrow(() -> new RuntimeException("用户不存在"));
  
    if (!passwordEncoder.matches(oldPassword, user.getPassword())) {
        throw new RuntimeException("旧密码不正确");
    }
  
    if (oldPassword.equals(newPassword)) {
        throw new RuntimeException("新密码不能与旧密码相同");
    }
  
    user.setPassword(passwordEncoder.encode(newPassword));
    userRepository.save(user);
}
```

**控制器接口**：


**安全效果**：

- ✅ 验证旧密码正确性，防止他人篡改密码
- ✅ 新密码必须符合复杂度要求
- ✅ 新密码不能与旧密码相同

### 2.2 登录安全

#### 2.2.1 登录失败次数限制与账户锁定

**安全准则**：防止暴力破解攻击，对多次登录失败的账户进行临时锁定。

**数据库设计**：

```java
// User.java - 新增字段（第47-52行）
@Column(name = "failed_attempts")
private Integer failedAttempts = 0;

@Column(name = "account_locked_until")
@Temporal(TemporalType.TIMESTAMP)
private Date accountLockedUntil;
```

**失败处理逻辑**：

```java
// CustomAuthenticationFailureHandler.java - 登录失败处理（第18-59行）
@Component
public class CustomAuthenticationFailureHandler implements AuthenticationFailureHandler {

    private static final int MAX_FAILED_ATTEMPTS = 5;
    private static final long LOCK_TIME_DURATION = 5 * 60 * 1000;  // 5分钟

    @Transactional
    private void incrementFailedAttempts(String username) {
        Optional<User> userOpt = userRepository.findByUsername(username);
        if (userOpt.isPresent()) {
            User user = userOpt.get();
            int newAttempts = user.getFailedAttempts() != null ? user.getFailedAttempts() + 1 : 1;
            user.setFailedAttempts(newAttempts);
          
            if (newAttempts >= MAX_FAILED_ATTEMPTS) {
                user.setAccountLockedUntil(new Date(System.currentTimeMillis() + LOCK_TIME_DURATION));
            }
            userRepository.save(user);
        }
    }
}
```

**登录时锁定检查**：

```java
// CustomUserDetailsService.java - 加载用户时检查锁定状态（第22-33行）
@Override
public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
    User user = userRepository.findByUsername(username)
            .orElseThrow(() -> new UsernameNotFoundException("用户名或密码错误"));
  
    if (user.getAccountLockedUntil() != null && user.getAccountLockedUntil().after(new Date())) {
        long remainingSeconds = (user.getAccountLockedUntil().getTime() - System.currentTimeMillis()) / 1000;
        throw new AuthenticationException("账户已被锁定，请 " + remainingSeconds + " 秒后再试") {};
    }
    return user;
}
```

**登录成功后重置**：

```java
// UserService.java - 登录成功后清除失败记录（第55-66行）
public Authentication login(LoginRequest request) {
    Authentication authentication = authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(request.getUsername(), request.getPassword()));
    SecurityContextHolder.getContext().setAuthentication(authentication);
  
    User user = userRepository.findByUsername(request.getUsername()).orElse(null);
    if (user != null && user.getFailedAttempts() != null && user.getFailedAttempts() > 0) {
        resetFailedAttempts(user);  // 清除失败记录
    }
  
    return authentication;
}
```

**安全效果**：

- ✅ 连续 5 次登录失败后账户锁定 5 分钟
- ✅ 显示剩余锁定时间提示
- ✅ 登录成功后自动重置失败计数
- ✅ 密码重置后自动解锁账户

### 2.3 身份认证（Authentication）

#### 2.3.1 Spring Security 框架集成

**安全准则**：使用成熟的安全框架处理认证流程，避免自行实现带来的安全漏洞。

**代码实现**：

```java
// SecurityConfig.java - 安全配置
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/", "/login", "/register", "/reset-password", "/css/**", "/js/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers("/api/documents/**").hasAnyRole("ADMIN", "USER")
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .loginProcessingUrl("/login/submit")
                .defaultSuccessUrl("/dashboard", true)
                .failureHandler(authenticationFailureHandler)
            )
            .logout(logout -> logout
                .logoutUrl("/logout")
                .logoutSuccessUrl("/login")
            );
        return http.build();
    }
}
```

**安全效果**：

- 自动处理登录表单验证和会话管理
- 基于角色的访问控制（RBAC）
- 自定义登录失败处理器，支持账户锁定提示

#### 2.3.2 JWT Token 认证

**安全准则**：无状态认证机制，支持 REST API 调用，避免会话劫持风险。

**代码实现**：

```java
// JwtTokenProvider.java - JWT 令牌生成与验证
@Component
public class JwtTokenProvider {
    @Value("${jwt.secret}")
    private String jwtSecret;
  
    @Value("${jwt.expiration}")
    private long jwtExpiration;

    public String generateToken(UserDetails userDetails) {
        return Jwts.builder()
                .claims(new HashMap<>())
                .subject(userDetails.getUsername())
                .issuedAt(new Date(System.currentTimeMillis()))
                .expiration(new Date(System.currentTimeMillis() + jwtExpiration))
                .signWith(getSigningKey())
                .compact();
    }
}
```

**安全效果**：

- 无需服务端存储会话，水平扩展友好
- Token 签名防篡改
- 过期时间自动失效

## 六、总结

本公文管理系统在安全方面实现了完整的安全防护体系：

### 核心安全措施

1. **认证安全**：Spring Security + JWT 双认证机制
2. **密码安全**：BCrypt 加密 + 自定义复杂度校验 + 修改验证旧密码
3. **登录安全**：5次失败锁定5分钟，自动解锁机制
4. **授权控制**：基于角色的访问控制（RBAC），最小权限原则
5. **输入验证**：Jakarta Validation 完整校验链
6. **数据安全**：JPA 参数化查询防 SQL 注入
7. **前端安全**：Thymeleaf 自动转义防 XSS
8. **配置安全**：环境变量管理敏感配置，避免硬编码

### 安全准则遵循

系统全面遵循 OWASP Top 10 安全准则，其中前8项已完全合规，剩余2项（A9、A10）为优化项，不影响核心安全性。

### 生产环境建议

在生产环境部署前，建议完成以下配置：

- 启用 MySQL SSL 连接
- 移除 `allowPublicKeyRetrieval=true` 参数
- 配置正式的 JWT 密钥和数据库密码
- 定期更新依赖版本
